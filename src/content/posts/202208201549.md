## TypeScript 简介

### 什么是 TypeScript

> Typed JavaScript at Any Scale.
> 添加了类型系统的 JavaScript，适用于任何规模的项目。

TypeScript 的两个最重要的特性——类型系统、适用于任何规模。

总结:

- TypeScript 是添加了类型系统的 JavaScript，适用于任何规模的项目。
- TypeScript 是一门静态类型、弱类型的语言。
- TypeScript 是完全兼容 JavaScript 的，它不会修改 JavaScript 运行时的特性。
- TypeScript 可以编译为 JavaScript，然后运行在浏览器、Node.js 等任何能运行 JavaScript 的环境中。
- TypeScript 拥有很多编译选项，类型检查的严格程度由你决定。
- TypeScript 可以和 JavaScript 共存，这意味着 JavaScript 项目能够渐进式的迁移到 TypeScript。
- TypeScript 增强了编辑器（IDE）的功能，提供了代码补全、接口提示、跳转到定义、代码重构等能力。
- TypeScript 拥有活跃的社区，大多数常用的第三方库都提供了类型声明。
- TypeScript 与标准同步发展，符合最新的 ECMAScript 标准（stage 3）。

#### 常用配置

- `"noEmitOnError": true`：类型检查错误不会编译成 js 文件
- `"target": "es5"`：向下兼容浏览器
- `"rootDir": "./src"`(在`/* Modules */`下)：定义源文件，也就是 ts 文件所在位置
- `"outDir": "./dist"`(在`/* Emit */`下)：定义输出文件，也就是 js 文件输出位置

### 回调中的可选参数

当你为回调写一个函数类型时，**永远不要写一个可选参数**，除非你打算在不传递该参数的情况下调用函数。

```typescript
function myForEach(arr: any[], callback: (arg: any, index?: number) => void) {
  for (let i = 0; i < arr.length; i++) {
    callback(arr[i], i);
  }
}
myForEach([1, 2, 3], (a, i) => {
  console.log(a, i);
});

myForEach([1, 2, 3], (a, i) => {
  console.log(i.toFixed()); // Object is possibly 'undefined'.
```

#### 编写好的重载

在可能的情况下，总是**倾向于使用联合类型的参数**而不是重载参数。

```typescript

function len(x: any) {
```
